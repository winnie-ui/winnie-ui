Goho includes a default set of colors that is a great place to start when building your next application. These colors are loosely based on the colors provided by [Tailwind](https://tailwindcss.com/docs/customizing-colors) and were created using the [Radix Colors](https://www.radix-ui.com/colors/custom) custom palette generator.

## Accent Color

Users and companies often have a preference for the accent color of their applications. Goho supports this using the `data-accent-color` attribute. The accent color attribute modifies the `--go-accent-color-<value>` css variables which will affect any components that use those variables.

```tsx title="root.tsx" ins="class="goho"" ins="data-accent-color="purple""
export default function Root() {
  return (
    <html>
      <head></head>
      <body class="goho" data-accent-color="purple">
        <App />
      </body>
    </html>
  );
}
```

Any color that Goho supports can be used as an accent color. Including a custom [brand color](/concepts/colors#branding).

## Customizing Colors

When the default palette doesn't suit your needs you can customize any of the colors by overriding the CSS Variables.

Luckily, the community has started to create some amazing tools for generating color palettes. Goho's color palette is based on the one defined by [Radix Colors](https://www.radix-ui.com/colors/custom). We suggest you start there when it comes to configuring your brand color.

Each color can be overridden using css variables defined in a global css file in your application. For example, if you want to define a custom `brand` color for your application you can do the following:

Once you have configured your custom brand color you can set `data-accent-color` to `brand` and all of your components will use that color as the accent.

```tsx title="root.tsx" ins="data-accent-color="brand""
export default function Root() {
  return (
    <html>
      <head></head>
      <body class="goho" data-accent-color="brand">
        <App />
      </body>
    </html>
  );
}
```
